## 🔄 Логика работы

### 1. Первый запуск пользователя
```
/start → Middleware → Проверка подписки → 
  ├─ НЕ подписан → Сообщение + кнопки [Подписаться] [Проверить]
  └─ Подписан → Выбор языка → Главное меню
```

### 2. Повторный запуск
```
/start → Middleware → Проверка подписки → 
  ├─ НЕ подписан → Блокировка доступа
  └─ Подписан → Главное меню (на выбранном языке)
```

### 3. Любое взаимодействие
```
Сообщение/Callback → Middleware → Проверка подписки →
  ├─ НЕ подписан → Блокировка + сообщение о подписке
  └─ Подписан → Обработка запроса
```

## 🔧 Технические детали

### Обработка ошибок
- ✅ Если пользователь заблокировал бота (рассылка)
- ✅ Если бот не админ канала (проверка подписки)
- ✅ Если канал не существует
- ✅ Сетевые ошибки (автоповтор)

### Middleware логика
```python
1. Получает событие (Message или CallbackQuery)
2. Извлекает user_id
3. Вызывает bot.get_chat_member(CHANNEL_ID, user_id)
4. Проверяет status in ['member', 'administrator', 'creator']
5. Если НЕ подписан → блокирует и показывает сообщение
6. Если подписан → продолжает обработку
```

### Исключения из проверки
- Callback "check_subscription" (кнопка "Я подписался")
